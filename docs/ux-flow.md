# 사내 근태 MVP UX Flow

작성일: 2026-07-08  
범위: 직원 모바일 화면, 관리자 운영 화면  
참고 문서: `outputs/DESIGN.md`, `outputs/internal-hr-attendance-prd.md`

## 1. UX 방향

### 1.1 제품 톤

- 직원 화면은 모바일 우선 업무 도구로 설계한다. 사용자가 출근, 퇴근, 휴가 신청 같은 핵심 행동을 10초 안에 끝낼 수 있어야 한다.
- 관리자 화면은 반복 운영에 맞춘 조밀한 정보 구조를 우선한다. 표, 필터, 상태 배지, 일괄 확인 액션을 중심으로 구성한다.
- Cal.com 스타일 레퍼런스를 반영해 장식 요소보다 명확한 계층, 단정한 흑백 팔레트, 둥근 액션 버튼, 얇은 구분선, 미세한 그림자만 사용한다.
- 핵심 CTA는 검정 배경의 pill 버튼으로 표현한다. 보조 액션은 흰색 또는 연회색 ghost pill 버튼을 사용한다.
- 상태는 텍스트와 배지를 함께 쓴다. 색상만으로 승인, 반려, 실패, 정정 상태를 구분하지 않는다.
- 급여와 위치 정보는 민감 정보로 간주한다. 접근 전후의 고지, 권한, 감사 로그가 UX 안에서 드러나야 한다.

### 1.2 공통 내비게이션

직원 모바일 하단 탭:

| 탭 | 목적 | 기본 진입 화면 |
|---|---|---|
| 홈 | 오늘의 출퇴근과 주요 알림 | 오늘 상태 |
| 근태 | 출퇴근 기록과 정정 요청 | 이번 주 기록 |
| 조기퇴근 | 월별/기간별 조기퇴근 조회 | 이번 달 요약 |
| 휴가 | 잔여 휴가와 신청 | 잔여/신청 |
| 야근 | 야근 신청과 승인 상태 | 신청 목록 |
| 급여 | 급여명세서 조회 | 월별 명세서 |
| 내 정보 | 개인정보, 근무지, 승인자 | 프로필 |

관리자 좌측 메뉴:

| 메뉴 | 목적 | 기본 진입 화면 |
|---|---|---|
| 대시보드 | 오늘 운영 현황 | 요약 카드와 승인 대기 |
| 직원 | 직원/권한/승인자 관리 | 직원 목록 |
| 근태 | 출퇴근 기록, 정정, 보정 | 기간별 근태 표 |
| 조기퇴근 | 조기퇴근 집계와 상태 관리 | 월별 직원 집계 |
| 휴가 승인 | 연차/반차/휴가 신청 처리 | 승인 대기 |
| 야근 승인 | 야근 신청과 실제 퇴근 비교 | 승인 대기 |
| 급여명세서 | 월별 파일 업로드와 열람 이력 | 업로드 현황 |
| 근무지/정책 설정 | GPS, QR, IP, 근무시간 정책 | 근무지 목록 |
| 감사 로그 | 민감 작업과 변경 이력 | 최근 로그 |

## 2. 직원 모바일 첫 화면

### 2.1 화면 목적

직원이 앱을 열자마자 오늘 무엇을 해야 하는지 알 수 있게 한다. 첫 화면은 랜딩 페이지가 아니라 실제 업무 화면이어야 한다.

### 2.2 기본 구성

상단:

- 사용자 이름, 부서, 오늘 날짜
- 근무지명과 위치 인증 상태
- 오늘의 상태 배지: `출근 전`, `근무 중`, `퇴근 완료`, `정정 필요`

메인 영역:

- 오늘 출근/퇴근 카드
- 현재 근무 시간 또는 다음 행동 안내
- 큰 1차 CTA: `출근하기` 또는 `퇴근하기`
- 보조 CTA: `QR로 인증`, `정정 요청`, `오늘 기록 보기`

요약 영역:

- 오늘 신청/승인 상태 한 줄 요약
- 이번 달 조기퇴근 누계
- 사용 가능 휴가 요약: 총 사용 가능, 법정 연차, 선사용 휴가
- 최신 급여명세서 공개 여부

### 2.3 상태별 첫 화면

| 상태 | 메인 CTA | 보조 정보 | 예외 액션 |
|---|---|---|---|
| 출근 전 | 출근하기 | 근무지, 현재 시각, 위치 권한 안내 | QR로 인증, 정정 요청 |
| 근무 중 | 퇴근하기 | 출근 시각, 현재 근무 시간, 오늘 신청 상태 | 정정 요청 |
| GPS 실패 | QR로 인증 | 실패 사유, 위치 권한 재시도 | GPS 실패 상태로 출퇴근 허용, 정정 요청 |
| 퇴근 완료 | 오늘 기록 보기 | 출근/퇴근 시각, 조기퇴근 여부 | 정정 요청 |
| 누락/오류 | 정정 요청 | 누락된 시각, 필요한 사유 | 기록 상세 |

### 2.4 수용 기준

- 직원은 홈 첫 화면에서 현재 출근 상태와 다음 액션을 즉시 확인할 수 있다.
- 출근 전에는 `출근하기`, 출근 후에는 `퇴근하기`가 가장 강한 CTA로 표시된다.
- GPS 실패 시 사용자는 같은 화면 흐름 안에서 QR 인증 또는 정정 요청으로 이동할 수 있다.
- 오늘의 휴가, 야근, 정정 승인 상태가 한 줄 요약으로 표시된다.
- 모바일 화면에서 핵심 CTA는 엄지로 누르기 쉬운 크기와 위치를 가진다.

## 3. 직원 출근/퇴근 흐름

### 3.1 출근 흐름

1. 직원이 홈에서 `출근하기`를 누른다.
2. 위치 권한이 없으면 브라우저 위치 권한 요청 전 안내 바텀시트를 보여준다.
3. 위치 권한을 허용하면 GPS 좌표와 정확도를 확인한다.
4. 등록 근무지 반경 안이면 출근 기록을 생성하고 완료 화면을 보여준다.
5. 반경 밖, 정확도 부족, 권한 거부, 수신 실패가 발생하면 실패 사유를 표시한다.
6. 직원은 `QR로 인증`, `다시 시도`, `정정 요청` 중 하나를 선택한다.
7. GPS 실패 상태에서 QR 또는 보조 버튼으로 출근하면 기록에 `GPS_FAILED_ALLOWED`를 남긴다.

완료 화면:

- 출근 시각
- 인증 방식: GPS, QR, Wi-Fi/IP 보조, GPS 실패 허용
- 근무지
- 오늘 다음 행동: `홈으로`, `근태 기록 보기`

### 3.2 퇴근 흐름

1. 직원이 홈에서 `퇴근하기`를 누른다.
2. 출근과 동일하게 GPS 인증을 우선 수행한다.
3. GPS 실패 시 QR, Wi-Fi/IP 보조, 정정 요청을 제공한다.
4. 퇴근 시각이 17:00 이전이면 조기퇴근 시간을 자동 계산한다.
5. 승인된 반차/휴가/조기퇴근 신청과 연결되면 해당 상태를 함께 표시한다.
6. 별도 승인 없이 17:00 이전 퇴근이면 `미승인 조기퇴근` 상태로 표시한다.
7. 퇴근 완료 후 오늘 근태 요약과 이번 달 조기퇴근 누계를 보여준다.

### 3.3 인증 실패와 정정 요청

정정 요청 진입 조건:

- 위치 권한 거부
- GPS 수신 실패 또는 정확도 부족
- 등록 근무지 반경 밖
- QR 인식 실패
- 긴급 외근, 출장, 현장 이동
- 출근/퇴근 누락

정정 요청 입력 항목:

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| 정정 유형 | 필수 | 출근 누락, 퇴근 누락, 시각 보정, 위치 예외 |
| 희망 시각 | 필수 | 직원이 요청하는 출근/퇴근 시각 |
| 사유 | 필수 | 승인자가 판단할 수 있는 설명 |
| 첨부/메모 | 선택 | 필요 시 증빙 또는 보조 설명 |

### 3.4 수용 기준

- 출근/퇴근 버튼을 누르는 순간에만 위치 인증을 요청한다.
- GPS 성공 시 직원은 별도 입력 없이 출근 또는 퇴근을 완료한다.
- GPS 실패 시 실패 사유와 대체 인증 방법이 명확히 표시된다.
- GPS 실패 후 QR 또는 보조 버튼으로 처리된 기록은 일반 GPS 성공 기록과 구분된다.
- 모든 정정 요청에는 사유와 희망 시각이 포함된다.
- 퇴근 시간이 17:00 이전이면 조기퇴근 시간이 자동 계산되고 상태가 지정된다.

## 4. 직원 휴가 신청 흐름

### 4.1 휴가 홈

휴가 탭의 첫 화면은 잔여 휴가와 신청 CTA를 함께 보여준다.

상단 요약:

- 사용 가능 휴가 합계
- 법정 연차 잔여
- 회사 선사용 휴가 가능분
- 승인 대기/승인/반려 신청 수

상세:

- 법정 연차 장부와 선사용 휴가 장부를 구분해 표시한다.
- 직원에게는 합산 사용 가능일을 먼저 보여주되, 상세 펼침에서 장부별 잔액과 상계 예정분을 확인할 수 있게 한다.

### 4.2 신청 흐름

1. 직원이 `휴가 신청`을 누른다.
2. 휴가 유형을 선택한다: 연차, 오전 반차, 오후 반차, 특별휴가, 무급휴가.
3. 날짜 또는 기간을 선택한다.
4. 시스템이 신청 일수와 잔여 한도를 즉시 계산한다.
5. 입사 첫해 직원이면 법정 연차 우선 차감 후 부족분이 선사용 휴가에서 차감됨을 표시한다.
6. 사유와 대체 인력 메모를 입력한다.
7. 중복 신청, 잔여 부족, 승인자 누락을 검증한다.
8. 제출 후 승인자에게 전달되고 직원은 신청 상세에서 상태를 확인한다.

### 4.3 신청 상태

| 상태 | 직원 표시 | 가능 액션 |
|---|---|---|
| 임시 저장 | 작성 중 | 수정, 삭제, 제출 |
| 승인 대기 | 승인자 검토 중 | 취소 요청 |
| 승인 | 휴가 확정 | 상세 보기 |
| 반려 | 반려됨 | 사유 확인, 다시 신청 |
| 취소 | 취소됨 | 상세 보기 |

### 4.4 수용 기준

- 직원은 휴가 신청 전에 사용 가능 휴가 합계와 법정/선사용 구분을 확인할 수 있다.
- 반차는 0.5일로 계산된다.
- 입사 3개월차 이후 직원은 회사 정책에 따라 선사용 휴가 신청이 가능하다.
- 법정 연차와 선사용 휴가는 화면 상세와 관리자 장부에서 분리되어야 한다.
- 잔여 부족, 날짜 중복, 승인자 미지정 상태에서는 제출할 수 없다.
- 승인된 휴가만 잔여 휴가에 반영된다.

## 5. 직원 조기퇴근 조회 흐름

### 5.1 화면 목적

직원이 본인의 조기퇴근 누계와 사유, 승인 상태를 월별 또는 기간별로 확인하게 한다. 급여 차감 여부를 단정하지 않고, 조회와 기록 투명성에 집중한다.

### 5.2 기본 구성

상단 요약:

- 이번 달 조기퇴근 총 시간
- 미승인 조기퇴근 시간
- 승인/휴가연동/유연운영 처리된 시간
- 선택 기간 필터

목록:

| 항목 | 표시 내용 |
|---|---|
| 날짜 | 근무일 |
| 퇴근 시각 | 실제 또는 보정 후 퇴근 시각 |
| 조기퇴근 시간 | 17:00 기준 계산값 |
| 상태 | 승인, 유연운영, 휴가연동, 미승인, 정정 |
| 연결 기록 | 반차, 휴가, 정정 요청, 관리자 보정 |

### 5.3 상세 화면

- 원본 퇴근 시각
- 보정 후 퇴근 시각
- 인증 방식
- 연결된 휴가/반차/정정 요청
- 승인자 또는 보정자
- 처리 사유

### 5.4 수용 기준

- 직원은 이번 달 조기퇴근 누계를 첫 화면에서 확인할 수 있다.
- 직원은 기간 필터로 본인 조기퇴근 내역을 조회할 수 있다.
- 조기퇴근 상태는 `승인`, `유연운영`, `휴가연동`, `미승인`, `정정`으로 구분된다.
- 직원은 조기퇴근 상세에서 원본 기록과 보정 또는 승인 연결 정보를 확인할 수 있다.
- 본인 데이터 다운로드 기능은 본인 기록만 포함한다.

## 6. 관리자 대시보드 흐름

### 6.1 화면 목적

관리자가 오늘 처리해야 할 승인, 정정, 보정, 급여 업로드 상태를 한 화면에서 파악하고 세부 화면으로 이동하게 한다.

### 6.2 기본 구성

상단 요약:

- 오늘 출근 현황
- 미퇴근 직원 수
- GPS 실패 허용 출퇴근 건수
- 승인 대기 휴가/야근/정정 요청 수
- 이번 달 미승인 조기퇴근 총 건수
- 급여명세서 업로드 진행률

작업 큐:

- 승인 대기 휴가
- 승인 대기 야근
- 정정 요청
- 보정 필요 후보
- GPS 실패 QR 출퇴근 내역

### 6.3 수용 기준

- 관리자는 대시보드에서 오늘 우선 처리할 업무를 확인할 수 있다.
- 각 요약 값은 해당 상세 목록으로 이동할 수 있다.
- GPS 실패 허용 기록은 별도 필터로 접근할 수 있다.
- 급여명세서 관련 카드에는 민감 정보 접근 경고와 권한 제한이 적용된다.

## 7. 관리자 승인 흐름

### 7.1 휴가 승인

1. 관리자가 `휴가 승인` 메뉴에 진입한다.
2. 기본 필터는 `승인 대기`다.
3. 목록에서 직원, 부서, 휴가 유형, 기간, 사용 일수, 잔여 현황, 사유를 확인한다.
4. 상세에서 법정 연차와 선사용 휴가 차감 예상, 중복 일정, 대체 인력 메모를 확인한다.
5. `승인` 또는 `반려`를 선택한다.
6. 반려 시 사유 입력이 필수다.
7. 승인 후 휴가 장부에 반영되고 감사 로그가 남는다.

### 7.2 야근 승인

1. 관리자가 `야근 승인` 메뉴에 진입한다.
2. 신청 날짜, 예상 시작/종료, 주간 연장근로 누계, 사유를 확인한다.
3. 1주 12시간 초과 가능성이 있으면 경고 배지를 표시한다.
4. 실제 퇴근 기록이 이미 있으면 신청 시간과 실제 퇴근 시간을 나란히 보여준다.
5. 승인 또는 반려를 처리한다.

### 7.3 정정 승인

1. 관리자가 `근태 > 정정 요청`에 진입한다.
2. 직원의 원본 기록, 요청 시각, 사유, 인증 실패 사유를 확인한다.
3. 필요 시 보정 후 시각을 조정한다.
4. 승인 또는 반려를 처리한다.
5. 승인 시 원본 기록은 삭제하지 않고 보정 이력을 남긴다.

### 7.4 수용 기준

- 관리자는 휴가, 야근, 정정 요청을 승인 또는 반려할 수 있다.
- 반려 시 사유 입력은 필수다.
- 휴가 승인 상세에는 법정 연차와 선사용 휴가 차감 예상이 표시된다.
- 야근 승인 상세에는 주간 연장근로 경고와 실제 퇴근 기록 비교가 표시된다.
- 모든 승인/반려 작업은 처리자, 처리 시각, 대상, 변경 내용을 감사 로그에 남긴다.

## 8. 관리자 근태 보정 흐름

### 8.1 진입 경로

- `근태 > 기간별 근태 표`
- `근태 > 정정 요청 상세`
- `조기퇴근 > 직원 상세`
- `직원 > 직원 상세 > 근태`

### 8.2 보정 유형

| 유형 | 설명 | 필수 입력 |
|---|---|---|
| 인정지각 | 지각 기록을 인정 처리 | 사유 |
| 인정조퇴 | 조기퇴근 기록을 인정 처리 | 사유 |
| 출근시각 보정 | 출근 시각 변경 | 변경 시각, 사유 |
| 퇴근시각 보정 | 퇴근 시각 변경 | 변경 시각, 사유 |
| 누락 기록 추가 | 출근 또는 퇴근 기록 생성 | 시각, 인증 대체 사유 |

### 8.3 보정 상세

보정 모달 또는 상세 패널에는 다음을 표시한다.

- 직원명, 부서, 근무일
- 원본 출근/퇴근 시각
- 원본 인증 방식과 실패 사유
- 변경할 시각 또는 상태
- 조기퇴근 재계산 결과
- 보정 사유 입력
- 감사 로그 저장 안내

### 8.4 수용 기준

- 관리자는 개인별 출퇴근 기록을 보정할 수 있다.
- 원본 기록은 삭제되지 않고 보정 전/후 값이 함께 보관된다.
- 보정 사유 없이는 저장할 수 없다.
- 보정 후 조기퇴근 시간이 변경되면 즉시 재계산되어 표시된다.
- 보정자, 보정 시각, 대상 직원, 원본 값, 변경 값, 사유가 감사 로그에 남는다.

## 9. 관리자 조기퇴근 조회 흐름

### 9.1 기본 화면

관리자는 월별 직원 집계와 기간별 상세를 오갈 수 있어야 한다.

필터:

- 기간
- 부서
- 직원
- 상태: 승인, 유연운영, 휴가연동, 미승인, 정정
- 인증 방식
- GPS 실패 허용 여부

집계 표:

| 열 | 설명 |
|---|---|
| 직원 | 이름, 부서 |
| 조기퇴근 총 시간 | 기간 내 총합 |
| 미승인 시간 | 별도 승인 없는 조기퇴근 |
| 승인/유연운영 시간 | 인정 처리된 시간 |
| 휴가연동 시간 | 반차/휴가와 연결된 시간 |
| 정정 건수 | 보정 또는 정정 요청 수 |
| 상세 | 직원별 상세 진입 |

### 9.2 수용 기준

- 관리자는 직원별 월별 조기퇴근 시간을 조회할 수 있다.
- 관리자는 부서와 기간으로 조기퇴근 데이터를 필터링할 수 있다.
- 승인된 조기퇴근과 미승인 조기퇴근은 명확히 구분된다.
- CSV 다운로드는 현재 필터 조건을 반영한다.
- 조기퇴근 조회 화면에서 인정조퇴 또는 유연운영 처리로 이동할 수 있다.

## 10. 관리자 급여명세서 흐름

### 10.1 업로드 흐름

1. 관리자가 `급여명세서` 메뉴에 진입한다.
2. 월을 선택한다.
3. 직원별 PDF 또는 이미지 파일을 업로드한다.
4. 시스템은 직원 매칭, 파일 누락, 중복 업로드를 검증한다.
5. 업로드 전 미리보기에서 직원명, 급여월, 파일명을 확인한다.
6. 관리자가 `공개`를 누르면 직원 화면에 노출된다.
7. 공개 이후 직원 열람, 다운로드 이력을 확인할 수 있다.

### 10.2 관리자 화면 구성

상단:

- 선택 월
- 업로드 완료 직원 수
- 미업로드 직원 수
- 공개 상태

목록:

| 열 | 설명 |
|---|---|
| 직원 | 이름, 부서 |
| 파일 상태 | 미업로드, 업로드 완료, 공개 |
| 업로드 일시 | HR 관리자 작업 시각 |
| 직원 열람 | 미열람, 열람 완료 |
| 다운로드 이력 | 횟수와 최근 시각 |
| 작업 | 업로드, 교체, 비공개, 상세 |

### 10.3 직원 조회 흐름

1. 직원이 `급여` 탭에 진입한다.
2. 월별 급여명세서 목록을 확인한다.
3. 명세서를 열기 전 민감 정보 안내를 확인한다.
4. 본인 명세서만 열람 또는 다운로드할 수 있다.
5. 열람과 다운로드는 감사 로그에 남는다.

### 10.4 수용 기준

- HR 관리자는 직원별 월별 급여명세서를 업로드할 수 있다.
- 공개 전에는 직원에게 명세서가 보이지 않는다.
- 직원은 본인 명세서만 조회할 수 있다.
- 열람과 다운로드 이력이 저장된다.
- 업로드, 교체, 공개, 비공개 작업은 감사 로그에 남는다.
- 관리자 화면은 월별 미업로드 직원과 미열람 직원을 쉽게 찾을 수 있어야 한다.

## 11. 화면별 공통 수용 기준

- 모바일 직원 주요 행동은 홈에서 2단계 이내에 완료된다.
- 관리자 목록 화면은 필터, 검색, 상태 배지, 정렬을 제공한다.
- 모든 민감 작업은 감사 로그 대상인지 사용자에게 알 수 있게 표현한다.
- 위치정보 수집은 출퇴근 인증 시점으로 제한한다는 안내가 제공된다.
- QR/Wi-Fi는 GPS 실패 보조 수단이며, 기록에는 인증 방식을 남긴다.
- 법정 연차, 선사용 휴가, 조정 장부는 관리자 화면에서 분리 조회된다.
- 상태 변경 후 직원과 승인자는 결과를 확인할 수 있다.
- 빈 상태, 로딩 상태, 오류 상태는 다음 행동을 함께 제시한다.

## 12. MVP 우선순위

1. 직원 모바일 홈, 출근, 퇴근, GPS 실패 후 QR/정정 흐름
2. 관리자 근태 목록, 정정 승인, 시간 보정
3. 휴가 잔여 조회, 휴가 신청, 관리자 승인
4. 조기퇴근 직원 조회, 관리자 월별/기간별 조회
5. 급여명세서 업로드, 공개, 직원 조회, 열람 로그
