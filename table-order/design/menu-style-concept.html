Table QR Order SaaS — Design Concept

메뉴판이 아니라,
매장의 매거진을 만든다

쇼핑몰식 상품 진열이 아니다. 전체 메뉴를 잡지 지면처럼 큐레이션하고, 메뉴 하나하나를 풀스크린 화보로 자세히 보여주며, 사진이 부족한 매장은 재료 입력만으로 AI 실사 연출컷을 얻는다. 그렇게 주문된 내역은 3초 안에 매장 관리자 화면에 울린다.

R1지면형 큐레이션 — 세로 타이포·인장·수식어 문구 R2화보 커버형 디테일 컷 — 대형 세리프 타이틀 R3AI 재료 분해컷 — 공중 부양 + 재료 라벨
예산QR 진입 → 주문 완료 ≤ 4탭 · 30초 예산옵션 메뉴 담기 ≤ 3탭 · 화면 전환 0회 예산첫 음식 사진 < 4초 · POS 알림 < 3초 예산추가 주문 '한 번 더' ≤ 3탭

※ 본 문서는 설계 시각화입니다. 실제 앱이 아니며, 사진 자리는 일러스트 플레이스홀더로 표현했습니다 — 실서비스에서는 매장 실사진과 AI 생성 실사 이미지가 들어갑니다. (근거 문서: docs/05, 06, 07, 12)

01

큐레이션 스프레드 — 전체 메뉴를 '지면'으로

레퍼런스 R1의 문법: 카테고리는 목록이 아니라 챕터다. 세로쓰기 표제 패널이 지면의 축이 되고, 메뉴 3~4개가 편집 리듬을 갖고 배치된다. 메뉴마다 수식어 → 이름 → 영문명 → 가격의 캡션 위계를 지킨다.

고객 화면 — /s/[매장]/… · 스크롤형 스프레드 피드

이 화면이 지키는 것

  • 챕터 표제부 — 세로쓰기(vertical-rl) 국문 타이포 + 지색 패널 + 인장(印) 모티프. 매장 테마로 on/off docs/05 §4.4 · §10 ThemeConfig
  • 캡션 위계(R1 문법) — 수식어 한 줄 → 큰 세리프 이름 → 영문 오너먼트 → 조용한 가격 docs/05 §4.4
  • 편집 배치 4종 + 밀도 규칙 — 지면당 2~4개, HERO는 챕터당 1회: 화보 리듬은 지키되 스크롤 폭발 방지. 음료·주류 챕터는 '인덱스 모드'로 압축 docs/05 §4.4
  • 퀵담기 시트 — ＋는 옵션이 있어도 화면 전환 없이 바텀시트에서 3탭 이내 종결. 사진/이름 탭만 디테일 컷 docs/05 §4.5
  • 차림 인덱스 — 헤더 '목차' 1탭 → 전 메뉴·가격 타이포 지면(급한 손님·재방문 fast lane, 바로 담기 가능) docs/05 §4.3
  • 다시 주문 — 주문 이력이 있으면 피드 상단 미니 카드 스트립 + 현황 화면 '한 번 더'(≤3탭) docs/05 §4.8
  • 내 테이블 신뢰 신호 — TABLE 7 배지 + 확인 시 "매장이 확인했어요 ✓" 승격 docs/02 §4 · docs/05 §4.8
  • 품절: 주문 시 서버가 즉시 거부(카트 자동 복구 2탭), 지면은 60초 내 반영 docs/05 §4.7 · §6
02

풀스크린 디테일 컷 — 주문할 음식을 '화보'로 본다

레퍼런스 R2의 문법: 메뉴를 탭하면 정보 패널이 아니라 요리책 커버 같은 풀스크린 화보가 열린다. 대형 세리프 타이틀, 연출컷 → 클로즈업 → 재료 분해컷 스와이프, 그리고 그 자리에서 옵션 선택·담기.

이 화면이 지키는 것

  • 커버 타이포존 — 기본은 국문 대제목(ko-first). R2식 영문 세리프 대제목("GALBI")은 매장이 명시 선택한 경우의 문법 docs/05 §4.6 titleLang
  • 컷 스와이프는 가로 — 연출컷 → 클로즈업 → AI 분해컷. 세로는 스토리 스크롤, 닫기는 최상단 pull-down — 제스처 축 분리로 오조작 방지 docs/05 §4.6
  • 담기 CTA는 첫 프레임부터 하단 고정 — 옵션 그룹은 스토리보다 위. 감상(스토리·재료)은 '더 읽기'로 선택 docs/05 §4.6
  • 스토리 문단 — 에디토리얼 인용 톤, 드롭캡 옵션. 재료 리스트는 알레르기 참고 겸 AI 연출컷 입력 MenuItem.story · ingredients
  • 주문 결정 정보(가격·CTA)는 16px·대비 7:1 하한 — 어두운 홀에서도 읽힌다 docs/05 §8
03

AI 연출컷 스튜디오 — 재료를 넣으면 화보가 나온다

레퍼런스 R3의 문법: 매장 운영자가 식재료를 디테일하게 입력하면, AI가 먹음직스러운 실사 연출 이미지(스테이징 히어로/재료 분해컷/클로즈업)를 생성한다. 재료 라벨은 이미지에 굽지 않고 데이터로 오버레이해 선명하고 수정 가능하게.

들기름 메밀 막국수 — 재료AI 생성

라벨은 이미지가 아니라 데이터(text, x, y) — 룩북 렌더러가 오버레이

운영자 플로우 — 6단계 (POS 메뉴 관리 안의 '스튜디오')

STEP 1
재료 입력
메뉴 폼의 재료 리스트 재사용 — "메밀면, 들기름(강조), 발효 간장…"
STEP 2
스타일 · 무드
히어로 연출 / 재료 분해 / 클로즈업 × 다크 우드 · 한지 라이트 · 스톤
STEP 3
생성
크레딧 1 차감 → 실사 후보 4장, p95 60초 이내
STEP 4
후보 비교 · 선택
4장 비교, 확대 검수. 전부 별로면 폐기
STEP 5
라벨 편집
분해컷 재료 라벨 자동 배치 → 드래그 조정·문구 수정
STEP 6
룩북 적용
변환(리사이즈·blurhash) 후 디테일 컷 '재료' 장에 편입

가드레일

검수 승인제 — 미검수 이미지 고객 노출 0 (불변식 I-8) 'AI 생성' 고지 배지 기본 ON 플랜별 월 크레딧 · 실패 자동 환불 텍스트는 이미지에 굽지 않음 — 라벨은 오버레이

설계 근거: docs/12 (파이프라인·프롬프트·크레딧), docs/05 §4 (라벨 렌더러), docs/03 AiImageJob

04

주문 → 매장 알림 → 조리 시작 — 3초의 파이프라인

룩북에서 전송된 주문은 실시간 채널로 매장 관리자 페이지(태블릿·내부 모니터링 PC·사장님 폰)에 알림음과 함께 도착한다. 직원이 확인을 누르는 순간 조리가 시작되고, 손님 화면의 타임라인이 함께 움직인다. 피크타임에는 일괄 확인(20건 3탭)·자동 접수 옵션, 주방에는 주방 모드 화면(읽기 전용 대형 보드)이 조리 시작점을 보장한다.

B-2주문 완료 ≤ 4탭 · 30초 A-1주문 → POS 카드+알림음 < 3초 A-8신규 20건 일괄 확인 ≤ 3탭 · 10초 B-6잠금·리로드 후 장바구니 복원 100%
