# 01. 제품 비전 & 요구사항

- 버전: v0.1 (2026-07-12)
- 지위: 설계 확정 문서 (변경 시 사유를 §8 변경이력에 기록)

## 1. 비전

**"모든 식당이 자기만의 매거진을 갖게 한다."**

기존 테이블오더(티오더·메뉴잇 류)는 기능적으로는 완성됐지만 경험적으로는 '창고형 쇼핑몰'이다 — 작은 썸네일, 획일적 그리드, 매장 개성 실종. 반면 브랜딩에 투자하는 매장(다이닝, 카페, 오마카세, 브런치, 와인바)은 음식 사진과 스토리가 '화보처럼' 보이길 원한다.

MENUBOOK은 메뉴판을 **패션 매거진의 룩북**으로 재정의한다: 풀블리드 사진, 에디토리얼 타이포그래피, 메뉴에 담긴 스토리. 손님은 '주문 단말기'가 아니라 '브랜드 경험'을 만나고, 매장은 객단가 상승(비주얼 어필 → 추가 주문)과 브랜드 자산을 얻는다. 동시에 운영은 실전 POS 수준으로: 실시간 주문 알림, 상태 관리, 테이블 세션 정산까지 한 화면에서.

## 2. 포지셔닝

| | 기존 테이블오더 | MENUBOOK |
|---|---|---|
| 메뉴 표현 | 썸네일 그리드, 텍스트 나열 | 풀블리드 화보, 스토리텔링, 편집 리듬 |
| 브랜딩 | 서비스사 UI에 매장이 끼워짐 | 매장별 커버/컬러/서체 테마 |
| 하드웨어 | 전용 태블릿 임대(테이블당 비용) | **손님 폰 QR** — 하드웨어 0원, 진입장벽 최소 |
| 관리자 | 별도 프로그램/전용기기 | 웹 POS(태블릿/PC 브라우저), 실시간 알림 |
| 과금 | 기기 임대료 중심 | SaaS 구독(플랜) 중심 |

**경쟁 구도(요약 — 상세·출처는 docs/13)**: 태블릿 진영(티오더·메뉴잇·페이히어 오더 등)은 통신사 결합 영업망과 광고 수익으로 단말 부담을 낮춰 중대형 매장에 강하고, 장기 약정(통상 수년, 불확실)이라 스위칭 대상이 아니다. 우리의 현실 타깃은 **태블릿 경제성이 안 나오는 8~20테이블 소형·브랜딩 매장 + 태블릿 미도입 신규 매장**이다. 비치헤드는 '인스타 브랜딩 투자형 브런치/베이커리 카페 × 단일 상권'(docs/13 §2).

**카피 방어 3선**: ① 가로 거치 공용 태블릿은 세로 4:5 풀블리드 '개인 폰 화보' 문법이 구조적으로 불리 ② 매장별 테마+AI 연출컷 이미지 자산은 축적형 락인 ③ 광고 수익 모델 진영은 '광고 없는 브랜드 지면'을 팔기 어렵다.

**MVP 비목표(명시)**: 기존 포스(포스뱅크 등) API 연동은 하지 않는다 — 결제·세무·현금영수증은 기존 포스가 계속 담당하고, 우리는 주문 접수와 일 마감 대사 리포트(docs/06 §6)로 병행 운영을 지원한다.

## 4. 유저 스토리 (ID는 이후 문서·테스트에서 참조)

### 고객 (C)
- **C-1** 손님은 테이블 QR을 스캔하면 로그인 없이 3초 내에 해당 매장·테이블로 식별된 메뉴판을 본다.
- **C-2** 손님은 매거진 커버→목차(카테고리)→화보형 피드로 메뉴를 탐색한다.
- **C-3** 손님은 메뉴 상세에서 사진 여러 장과 스토리, 옵션(필수/선택)을 보고 장바구니에 담는다.
- **C-4** 손님은 장바구니에서 수량 조정 후 주문을 전송하고, 주문 접수 확인을 즉시 받는다.
- **C-5** 손님은 주문 현황(접수→확인→조리중→서빙완료)을 실시간으로 본다.
- **C-6** 손님은 같은 세션에서 추가 주문을 할 수 있고, 누적 주문 내역·합계를 본다.
- **C-7** 손님은 직원 호출/계산서 요청 버튼을 쓸 수 있다.
- **C-8** (선결제 매장) 손님은 주문 시 토스페이먼츠로 결제한다.
- **C-9** 품절 메뉴는 피드에서 명확히 표시되고 주문할 수 없다.
- **C-10** 손님은 메뉴를 탭하면 **풀스크린 디테일 컷 화보**(연출컷·클로즈업·AI 분해컷)로 주문할 음식을 자세히 살펴본다.
- **C-11** 손님은 주문 현황에서 이전 주문 항목을 `한 번 더` **3탭 이내**로 재주문한다(옵션 스냅샷 재사용).
- **C-12** 손님은 정산 후에도 24시간 동안 읽기 전용 영수증 뷰로 주문 내역을 열람한다.
- **C-13** 손님은 옵션이 있는 메뉴도 화면 전환 없이 퀵담기 시트에서 **3탭 이내**로 담는다.

### 매장 (S)
- **S-1** 오너는 이메일로 가입하고 온보딩 위저드로 매장을 개설한다(정보→테이블→QR→메뉴→테마).
- **S-2** 오너는 테이블을 등록하면 테이블별 QR(인쇄용 PDF 포함)을 발급받는다.
- **S-3** 오너는 카테고리/메뉴/사진(다중)/옵션그룹을 관리하고, 룩북 미리보기로 결과를 즉시 확인한다.
- **S-4** 오너는 매장 테마(커버 이미지, 컬러, 서체 무드)를 설정한다.
- **S-5** 직원은 POS 주문 보드에서 신규 주문을 **알림음과 함께 3초 내** 수신한다.
- **S-6** 직원은 주문을 확인(수락)하거나 사유와 함께 거절하고, 조리중/서빙완료로 상태를 바꾼다.
- **S-7** 직원은 품절 토글을 즉시 반영할 수 있다(손님 화면에도 실시간 반영).
- **S-8** 직원은 테이블 세션의 누적 계산서를 보고 카운터 결제(현금/카드)를 기록하며 세션을 닫는다.
- **S-9** 오너는 일/주/월 매출과 인기 메뉴 통계를 본다.
- **S-10** 오너는 직원 계정(역할: OWNER/MANAGER/STAFF)을 관리한다.
- **S-11** 직원 호출 요청이 POS에 실시간 표시된다.
- **S-12** 오너는 영업시간 외 주문 차단, 선결제 필수 여부 등 운영 옵션을 설정한다.
- **S-13** 오너는 메뉴의 식재료·구성을 디테일하게 입력하면 **AI가 먹음직스러운 실사 연출컷**(스테이징 히어로 컷/재료 분해컷/클로즈업)을 생성해 주고, 후보 중 선택·검수하여 룩북에 적용한다.
- **S-14** 오너는 분해컷 위 재료 라벨(문구·위치)을 편집한다.
- **S-15** 주방은 확정 주문을 **주방 모드 화면(또는 주방 프린터)**으로 홀을 거치지 않고 확인한다.
- **S-16** 직원은 진행 중 세션을 다른 테이블로 이동한다(주문·합계 유지, 손님 화면 무중단).
- **S-17** 오너는 일 마감 리포트로 기존 포스 시재와 5분 내 대사한다.

### 플랫폼 (P)
- **P-1** 방문자는 랜딩에서 데모 매장 룩북을 체험하고 가입한다.
- **P-2** 매장은 14일 무료 체험 후 플랜(BASIC/PRO)을 구독한다(토스 빌링).
- **P-3** 플랜 한도(테이블 수·메뉴 수·기능)가 초과되면 업그레이드 안내가 뜬다.
- **P-4** 슈퍼어드민은 전체 매장 목록/상태/지표를 보고 매장을 정지시킬 수 있다.
- **P-5** 구독 결제 실패 시 유예기간(dunning) 후 서비스가 제한된다.

## 6. 비기능 요구사항 (NFR)

| ID | 항목 | 목표 |
|---|---|---|
| N-1 | 룩북 초기 로드 | **p75 LCP < 1.8s**(Slow 4G), TTFB < 600ms, QR 인식→첫 음식 사진 가시 p75 < 4s, INP < 200ms — 상세 예산 docs/05 §8 |
| N-2 | 주문→POS 알림 지연 | p95 < 3s (실시간 채널), 폴백 폴링 시 < 10s |
| N-3 | 테넌트 격리 | 모든 데이터 접근은 storeId 스코프 강제. 교차 테넌트 접근 시 403. 회귀 테스트로 보증 |
| N-4 | 가용성 | 주문 API 이중 안전장치: 실시간 채널 장애 시에도 주문 저장은 성공하고 POS 폴링으로 수신 |
| N-5 | 보안 | 고객 표면은 무인증이지만 tableToken(추측 불가 랜덤)으로만 주문 가능, rate limit 적용. QR 사진 원격 주문 방어: 새 손님 첫 주문 뱃지·세션 첫 주문 수동 확인·원탭 토큰 회전(docs/06 §3~4) |
| N-6 | 접근성·터치 | 고객·POS 공통 히트영역 ≥ 44×44px(CTA 48px). 주문 결정 정보(가격·품절·CTA) ≥ 16px·대비 7:1, 장식 텍스트 ≥ 12px·4.5:1 — 상세 docs/05 §8 |
| N-10 | 인터랙션 예산 | QR 진입→주문 완료 ≤ 4탭·30초, 옵션 메뉴 담기 ≤ 3탭·전환 0회, 추가 주문 ≤ 3탭 — 전체 표 docs/05 §2 (qa E2E로 강제) |
| N-11 | 브라우저 매트릭스 | iOS Safari / Chrome Android / 삼성 인터넷 / 카카오톡·네이버·인스타그램 인앱 6종에서 진입→담기→주문(선결제 포함) 스모크 통과 (M2·M4 DoD) |
| N-7 | 규모 가정(MVP) | 동시 접속 매장 200, 매장당 동시 테이블 30, 피크 주문 10건/분/매장 |
| N-8 | 데이터 보존 | 주문/결제 5년, 감사로그 1년. 메뉴 삭제는 soft delete(주문 스냅샷 무결성) |
| N-9 | AI 연출컷 | 생성 잡 p95 < 60s(후보 4장), **미검수 AI 이미지의 고객 노출 0**(승인제 — 회귀 테스트로 보증), 플랜별 월 크레딧으로 비용 통제 |

## 7. 성공 지표 (North Star: 월 활성 주문 수)

- 활성화: 가입→첫 실주문 도달률 ≥ 40%(중앙값 ≤ 2일), 온보딩 완주(메뉴 10개+사진 등록) ≥ 60%
- 경험: QR 스캔→첫 주문 전환율 ≥ 55%, 세션당 추가주문율 ≥ 25% — 설계 예산은 N-10/docs/05 §2가 담보
- 리텐션: 매장 월 이탈률 < 5%, 체험→유료 전환 ≥ 25%, 무료→유료 전환 ≥ 8%/90일, 선행지표: 월 1회 이상 통계 조회 오너 ≥ 70%
