# 06. POS 관리자 UX/UI 설계 (매장 표면)

- 버전: v0.1 (2026-07-12)
- 구현 소유: `pos-ui` 에이전트 (`apps/web/app/(admin)`), 실시간 수신은 `realtime` 훅 사용
- 1차 디바이스: 태블릿 가로(1024×768+) 브라우저 상시 거치. 내부 모니터링 **PC**와 **사장님 폰(모바일 웹)** 반응형 지원 — 폰에서도 신규 주문 알림 수신·주문 확인이 가능해야 한다.

## 1. 설계 원칙

1. **놓치지 않는다** — 신규 주문은 소리+색+배지 삼중 신호. 화면 어디에 있어도 인지 가능(글로벌 알림 레이어).
2. **두 번 탭 이내** — 피크타임 기준 핵심 동작(주문 확인, 서빙 완료, 품절 토글)은 보드에서 두 번 탭 이내.
3. **큰 타깃, 명확한 상태색** — 터치 타깃 ≥ 44px. 상태 컬러: PENDING=주황(점멸 급), CONFIRMED=파랑, IN_PROGRESS=보라, SERVED=초록, 경과시간 초과=빨강 링.
4. **셀프서비스 관리** — 메뉴/테이블/테마 관리는 CS 없이 오너 혼자 완료 가능해야 한다(룩북 미리보기 내장).
5. **주방까지 닿아야 주문이다** — 홀 태블릿에만 뜨는 주문은 절반의 전달이다. 주방 모드(§3)와 주방 프린터(로드맵)로 조리 시작점을 보장한다.
6. **기존 포스와 싸우지 않는다** — 결제·세무·현금영수증은 매장의 기존 포스가 계속 담당한다. 우리는 '주문 접수'를 맡고, 정산 대사를 5분 내 끝낼 리포트를 제공한다(§6).

## 2. 정보 구조 (좌측 아이콘 레일 내비)

```
/s/[slug]/admin
├── (주문 보드)      ← 기본 화면, 상주
├── tables/          ← 테이블맵·세션·QR
├── menu/            ← 카테고리·메뉴·옵션·사진 (+룩북 미리보기)
├── kitchen/         ← 주방 모드(읽기 전용 대형 보드 — 주방 거치용)
├── checkout/        ← 정산 이력·영수증·일 마감
├── stats/           ← 매출·인기 메뉴·객단가
└── settings/        ← 매장 정보·테마·운영 옵션·직원·구독
```

상단 글로벌 바: 매장명 | 오늘 매출·주문수 요약 | 실시간 연결 상태 표시등(●연결/○폴백) | 호출 벨 아이콘(미처리 수) | 주문 일시정지 토글(비상용, `settings.orderPauseUntil`).

## 3. 주문 보드 (기본 화면)

- **칸반 3열**: `신규(PENDING)` | `진행중(CONFIRMED/IN_PROGRESS)` | `완료(SERVED, 최근 20건)`. 태블릿 세로/폰은 탭 전환.
- 주문 카드:
  ```
  ┌──────────────────────────────┐
  │ No.14 · 테이블 7        03:12 │  ← orderNo, 테이블, 경과시간(1분 단위, 5분↑ 빨강)
  │ 트러플 리조또 ×2  (샷 추가)    │
  │ 수제 맥주 ×1                  │
  │ 메모: 고수 빼주세요 ⚠         │  ← 메모는 항상 강조
  │ ₩25,000     [거절]  [확인 ✓] │
  └──────────────────────────────┘
  ```
- 액션: 신규 카드 `확인`(→CONFIRMED) / `거절`(사유 선택: 품절·마감·**해당 테이블 손님 아님**·기타+텍스트, 품절 선택 시 해당 메뉴 품절 토글 제안, '손님 아님' 선택 시 해당 세션 즉시 종료+쿠키 무효화 제안). 진행중 카드 `조리 시작`(설정 시) / `서빙 완료`. 실행 취소 스낵바 5초.
- **피크타임 처리 (수용 기준 A-8)**: 신규 열 상단 `전체 확인 (N건)` 일괄 버튼(STAFF, 실행 취소 유지). 신규 열 정렬은 오래된 순 고정 + 5분 초과 카드 자동 최상단 고정. `settings.autoConfirmOrders`=ON이면 주문 생성 시 즉시 CONFIRMED(actor=SYSTEM, AuditLog) + 인쇄/알림 발화 — 이때 손님의 PENDING 취소 창이 사라지므로 룩북 취소 버튼은 '취소 요청(호출 kind=CUSTOM)'으로 대체된다.
- **원격 장난 주문 방어**: 직전 세션 CLOSED 후 첫 진입으로 열린 세션의 첫 주문 카드에 `새 손님 첫 주문` 뱃지 + 구분 알림음 + 테이블맵 해당 테이블 점멸(직원이 착석 여부를 눈으로 1초 대조). `settings.firstOrderManualConfirm`(기본 ON): 자동접수 매장이라도 세션 첫 주문만은 수동 확인.
- **신규 주문 수신 UX**: 알림음(Web Audio) + 카드 하이라이트 + 좌측 레일 배지 + (권한 허용 시) 브라우저 Notification. 재알림은 카드별 30s 반복이 아니라 **'미확인 존재' 통합 신호** — 1분 경과 미확인이 있으면 연속음 모드(기본 ON). 사운드는 사용자 제스처 후 활성화되는 제약 안내 배너 처리.
- 직원 호출 카드: 보드 상단 별도 스트립(주황 계열, `테이블 7 · 직원 호출 · 00:41`) — `완료` 원탭. **동일 테이블·동일 종류의 반복 호출은 새 카드 대신 기존 카드에 `×2` 카운트 병합**(알림음 재발화 1회만), OPEN 호출 10분 자동 만료. 세션 패널에서 `이 테이블 호출 일시 차단(30분)` 토글(손님 화면에는 '직원이 곧 갑니다'로 표기).
- **주방 모드(kitchen/)**: 주문 보드의 읽기 전용 변형 — CONFIRMED 이후만 표시, 카드당 화면 1/3 크기 대형 텍스트(3m 가독), `조리 완료` 원탭. 프린터 없는 매장이 주방 중고 태블릿에 STAFF 계정으로 거치하는 용도. 주방 전달 수단(프린터/주방 모드) 미구성 매장은 손님 접수 문구를 '주방에 전달됐어요' 대신 '매장에 전달됐어요'로 자동 조정.
- 연결 상태: 실시간 채널 단절 시 상단 표시등 폴백 전환 + `sinceEventAt` diff 폴링 5s로 자동 강등, 복구 시 재동기화(docs/07 §4).

## 4. 테이블 & 세션 & QR (`tables/`)

- **테이블맵 그리드 — 상태 3단계**: 빈 테이블 / **열람중**(세션 OPEN·주문 0건, 회색 점 — QR만 찍고 간 유령 세션과 구분) / **이용중**(첫 주문 발생부터, 경과시간·누적금액·계산요청 점멸). 주문 0건 세션은 마지막 활동 30분 후 자동 정리(`EXPIRED` — 정산 불변식 I-5와 무관).
- 테이블 탭 → 세션 패널: 누적 주문 라인아이템(주문별 그룹 + **시각·기기 뱃지 폰①/폰②** — 분쟁 증빙, 기기 3대 이상이면 강조), 합계, `정산하기` / `테이블 이동(STAFF)` / `QR 재발급(의심 시 원탭 회전)` / `호출 일시 차단` / `세션 강제종료(MANAGER)`.
- **테이블 이동**: 빈 테이블 선택 → `TableSession.tableId` 재바인딩(기존 Order.tableId는 이력 보존, AuditLog 기록). 손님 쿠키는 sessionId 기준이라 무변경 — 목적지 테이블 QR을 다시 찍어도 OPEN 세션에 합류한다. (합석·계산 분할은 백로그 유지)
- **정산 플로우**: `정산하기` → 결제수단 선택(현금/카드단말) → 금액 확인(기본 전액) → 기록 → 완납 시 세션 CLOSE + SERVED→COMPLETED 일괄 전이 + 테이블 초기화. 선결제 주문은 라인에 `결제완료` 뱃지로 구분되어 정산 합계에서 자동 제외.
- **기존 포스 병행 운영(현실 기본값)**: 정산 패널에 '기존 포스 입력용 요약' 블록 — 합계 금액 초대형 표기 + 메뉴별 수량 요약 `복사` 버튼(금액만 찍는 매장/메뉴별 찍는 매장 모두 대응). 현금 결제 기록 시 "현금영수증은 기존 단말에서 발급해 주세요" 안내 고정. 결제수단 기록을 생략하는 매장용 `settings.skipPaymentRecord` 토글(주문 접수만 쓰고 정산은 전부 기존 포스로 하는 매장) — 이 경우 세션 CLOSE는 금액 확인 없이 원탭.
- 테이블 CRUD: 라벨·정렬, 생성 시 qrToken 자동 발급. `QR 인쇄` → 전 테이블 QR 시트 PDF(A4, 테이블당 카드형 — 매장 로고·"QR을 찍고 매거진을 펼치세요" 문구). 토큰 회전(재발급) 시 기존 QR 즉시 무효 경고.

## 5. 메뉴 관리 (`menu/`) — 룩북의 백오피스

- 2패널: 좌측 카테고리(드래그 정렬) / 우측 해당 메뉴 리스트(드래그 정렬, 품절·숨김 토글 스위치 인라인).
- 메뉴 편집 폼: 이름·summary·story(마크다운 에디터, 드롭캡 미리보기)·가격·배지·layoutHint·**재료/구성 리스트(ingredients — 디테일 컷 표기 겸 AI 연출컷 입력)**·옵션그룹 빌더(그룹: min/max, 선택지: 이름/±가격/품절).
- 사진 업로더: 다중 업로드, 드래그 정렬(1번이 대표), 업로드 후 자동 변환 상태 표시. 권장 가이드 노출: "4:5 세로, 2000px 이상, 자연광".
- **AI 연출컷 스튜디오**(docs/12): 재료 리스트 기반 생성 — 스타일(스테이징 히어로/재료 분해/클로즈업)·무드(다크 우드/한지 라이트/스톤 등)·강조 재료 선택 → `생성`(크레딧 1) → 후보 4장 비교 → 선택. 분해컷은 **라벨 에디터**(자동 초기 배치 후 드래그 조정)로 재료 라벨 확정 → 대표컷 지정. 잔여 크레딧 게이지 + 업그레이드 CTA. 미검수 후보에는 '아직 고객에게 보이지 않음' 상태 표기(I-8).
- **룩북 미리보기(차별 기능)**: 우측 슬라이드오버에 실제 룩북 렌더(모바일 프레임) — 편집 내용 즉시 반영. 저장 전 draft 미리보기.
- 품절 토글은 STAFF도 가능(보드에서 접근하는 빠른 경로 겸용), 그 외 편집은 MANAGER↑.

## 6. 정산 이력 (`checkout/`) & 통계 (`stats/`)

- 정산 이력: 세션 단위 리스트(일자 필터) — 테이블, 시간, 주문수, 금액, 결제수단 구성. 상세에서 주문·결제 라인 + PG 영수증 링크.
- **일 마감 리포트**: 영업일(`settings.businessDayCutoff`, 기본 06:00) 기준 현금/카드/PG 구분 합계 + 건수 + 미정산 세션 목록 + 강제종료 목록 — **기존 포스 시재점검과 5분 내 대사** 가능한 형태(인쇄/CSV).
- 통계(OWNER): 기간 선택(오늘/7일/30일) — MVP 5개 위젯 고정: 매출 추이, 시간대 히트맵, 인기 메뉴 Top 10, 테이블 회전율(**주문 1건 이상 세션만 집계** — 열람중 세션 제외), **세션당 평균 객단가 + 추가주문 기여액**("룩북이 만든 추가 주문 ₩X" — 세션 내 2번째 이후 주문 합계. 구독료 ROI를 매달 숫자로 증명하는 위젯).
- 월 1회 오너 이메일 리포트(빌링 크론 인프라 재사용): 주문수·객단가·추가주문율·AI 연출컷 적용 메뉴의 적용 전후 2주 변화.

## 7. 설정 (`settings/`)

| 탭 | 내용 |
|---|---|
| 매장 정보 | 이름·소개·전화·주소, slug(변경 시 QR 재인쇄 경고, OWNER) |
| 룩북 테마 | ThemeConfig 편집(docs/05 §10) + 커버 이미지 업로드 + 실시간 미리보기 |
| 운영 | 영업시간(요일별), 영업일 경계(`businessDayCutoff`, 기본 06:00), 주문 일시정지, `useInProgressStatus`(조리중 단계), `prepayRequired`(선결제 필수), `autoConfirmOrders`(자동 접수), `firstOrderManualConfirm`(세션 첫 주문 수동 확인, 기본 ON), `skipPaymentRecord`(기존 포스 정산 위임), 미확인 연속음, 미확인 에스컬레이션(X분 경과 시 지정 번호 SMS — 베타 옵션, 프린터 매장 기본 OFF) |
| 직원 | 초대(이메일)·역할 변경·비활성화 (OWNER) |
| 구독 | 현재 플랜·한도 사용량 게이지·결제수단·플랜 변경 (OWNER, docs/09) |

## 8. 태블릿 상주 모드

- PWA(manifest + 홈 추가 안내), Wake Lock API로 화면 꺼짐 방지 토글, 전체화면 모드.
- **개점 사운드 체크**: 영업일 첫 로그인 시 배너의 `사운드 체크` 버튼 — Web Audio 제스처 확보 + 볼륨 실측을 한 번에(알림음이 안 울리는 채로 영업 시작하는 사고 방지). iPadOS/Android 브라우저별 백그라운드 제약과 권장 거치 설정은 운영 가이드 문서(M6 산출물)에 명기.
- 세션 만료 대응: 스태프 세션은 장기 리프레시(30일), 401 시 PIN 재잠금 화면(백로그) 대신 MVP는 재로그인 유도.
- 오프라인: 배너 표시 + 조작 차단(낙관적 업데이트로 인한 유실 방지), 복구 시 보드 전체 재동기화.

## 9. 수용 기준 하이라이트 (qa 시나리오 씨앗)
