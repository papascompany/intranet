# 06. POS 관리자 UX/UI 설계 (매장 표면)

- 버전: v0.1 (2026-07-12)
- 구현 소유: `pos-ui` 에이전트 (`apps/web/app/(admin)`), 실시간 수신은 `realtime` 훅 사용
- 1차 디바이스: 태블릿 가로(1024×768+) 브라우저 상시 거치. 내부 모니터링 **PC**와 **사장님 폰(모바일 웹)** 반응형 지원 — 폰에서도 신규 주문 알림 수신·주문 확인이 가능해야 한다.

## 1. 설계 원칙

1. **놓치지 않는다** — 신규 주문은 소리+색+배지 삼중 신호. 화면 어디에 있어도 인지 가능(글로벌 알림 레이어).
2. **두 번 탭 이내** — 피크타임 기준 핵심 동작(주문 확인, 서빙 완료, 품절 토글)은 보드에서 두 번 탭 이내.
3. **큰 타깃, 명확한 상태색** — 터치 타깃 ≥ 44px. 상태 컬러: PENDING=주황(점멸 급), CONFIRMED=파랑, IN_PROGRESS=보라, SERVED=초록, 경과시간 초과=빨강 링.
4. **셀프서비스 관리** — 메뉴/테이블/테마 관리는 CS 없이 오너 혼자 완료 가능해야 한다(룩북 미리보기 내장).

## 2. 정보 구조 (좌측 아이콘 레일 내비)

```
/s/[slug]/admin
├── (주문 보드)      ← 기본 화면, 상주
├── tables/          ← 테이블맵·세션·QR
├── menu/            ← 카테고리·메뉴·옵션·사진 (+룩북 미리보기)
├── checkout/        ← 정산 이력·영수증
├── stats/           ← 매출·인기 메뉴
└── settings/        ← 매장 정보·테마·운영 옵션·직원·구독
```

상단 글로벌 바: 매장명 | 오늘 매출·주문수 요약 | 실시간 연결 상태 표시등(●연결/○폴백) | 호출 벨 아이콘(미처리 수) | 주문 일시정지 토글(비상용, `settings.orderPauseUntil`).

## 3. 주문 보드 (기본 화면)

- **칸반 3열**: `신규(PENDING)` | `진행중(CONFIRMED/IN_PROGRESS)` | `완료(SERVED, 최근 20건)`. 태블릿 세로/폰은 탭 전환.
- 주문 카드:
  ```
  ┌──────────────────────────────┐
  │ No.14 · 테이블 7        03:12 │  ← orderNo, 테이블, 경과시간(1분 단위, 5분↑ 빨강)
  │ 트러플 리조또 ×2  (샷 추가)    │
  │ 수제 맥주 ×1                  │
  │ 메모: 고수 빼주세요 ⚠         │  ← 메모는 항상 강조
  │ ₩25,000     [거절]  [확인 ✓] │
  └──────────────────────────────┘
  ```
- 액션: 신규 카드 `확인`(→CONFIRMED) / `거절`(사유 선택: 품절·마감·기타+텍스트, 품절 선택 시 해당 메뉴 품절 토글 제안). 진행중 카드 `조리 시작`(설정 시) / `서빙 완료`. 실행 취소 스낵바 5초.
- **신규 주문 수신 UX**: 알림음(Web Audio, 확인 전까지 30s 간격 재알림 옵션) + 카드 하이라이트 + 좌측 레일 배지 + (권한 허용 시) 브라우저 Notification. 사운드는 사용자 제스처 후 활성화되는 제약 안내 배너 처리.
- 직원 호출 카드: 보드 상단 별도 스트립(주황 계열, `테이블 7 · 직원 호출 · 00:41`) — `완료` 원탭.
- 연결 상태: 실시간 채널 단절 시 상단 표시등 폴백 전환 + `sinceEventAt` diff 폴링 5s로 자동 강등, 복구 시 재동기화(docs/07 §4).

## 4. 테이블 & 세션 & QR (`tables/`)

- **테이블맵 그리드**: 테이블 카드 = 라벨 + 세션 상태(빈 테이블/이용중·경과시간/계산요청 점멸) + 세션 누적금액.
- 테이블 탭 → 세션 패널: 누적 주문 라인아이템(주문별 그룹), 합계, `정산하기` / `세션 강제종료(MANAGER)`.
- **정산 플로우**: `정산하기` → 결제수단 선택(현금/카드단말) → 금액 확인(기본 전액) → 기록 → 완납 시 세션 CLOSE + SERVED→COMPLETED 일괄 전이 + 테이블 초기화. 선결제 주문은 라인에 `결제완료` 뱃지로 구분되어 정산 합계에서 자동 제외.
- 테이블 CRUD: 라벨·정렬, 생성 시 qrToken 자동 발급. `QR 인쇄` → 전 테이블 QR 시트 PDF(A4, 테이블당 카드형 — 매장 로고·"QR을 찍고 매거진을 펼치세요" 문구). 토큰 회전(재발급) 시 기존 QR 즉시 무효 경고.

## 5. 메뉴 관리 (`menu/`) — 룩북의 백오피스

- 2패널: 좌측 카테고리(드래그 정렬) / 우측 해당 메뉴 리스트(드래그 정렬, 품절·숨김 토글 스위치 인라인).
- 메뉴 편집 폼: 이름·summary·story(마크다운 에디터, 드롭캡 미리보기)·가격·배지·layoutHint·**재료/구성 리스트(ingredients — 디테일 컷 표기 겸 AI 연출컷 입력)**·옵션그룹 빌더(그룹: min/max, 선택지: 이름/±가격/품절).
- 사진 업로더: 다중 업로드, 드래그 정렬(1번이 대표), 업로드 후 자동 변환 상태 표시. 권장 가이드 노출: "4:5 세로, 2000px 이상, 자연광".
- **AI 연출컷 스튜디오**(docs/12): 재료 리스트 기반 생성 — 스타일(스테이징 히어로/재료 분해/클로즈업)·무드(다크 우드/한지 라이트/스톤 등)·강조 재료 선택 → `생성`(크레딧 1) → 후보 4장 비교 → 선택. 분해컷은 **라벨 에디터**(자동 초기 배치 후 드래그 조정)로 재료 라벨 확정 → 대표컷 지정. 잔여 크레딧 게이지 + 업그레이드 CTA. 미검수 후보에는 '아직 고객에게 보이지 않음' 상태 표기(I-8).
- **룩북 미리보기(차별 기능)**: 우측 슬라이드오버에 실제 룩북 렌더(모바일 프레임) — 편집 내용 즉시 반영. 저장 전 draft 미리보기.
- 품절 토글은 STAFF도 가능(보드에서 접근하는 빠른 경로 겸용), 그 외 편집은 MANAGER↑.

## 6. 정산 이력 (`checkout/`) & 통계 (`stats/`)

- 정산 이력: 세션 단위 리스트(일자 필터) — 테이블, 시간, 주문수, 금액, 결제수단 구성. 상세에서 주문·결제 라인 + PG 영수증 링크. CSV 내보내기(백로그).
- 통계(OWNER): 기간 선택(오늘/7일/30일) — 매출 추이 라인, 시간대 히트맵, 인기 메뉴 Top 10(주문수·매출), 테이블 회전율. MVP는 이 4개 위젯 고정.

## 7. 설정 (`settings/`)

| 탭 | 내용 |
|---|---|
| 매장 정보 | 이름·소개·전화·주소, slug(변경 시 QR 재인쇄 경고, OWNER) |
| 룩북 테마 | ThemeConfig 편집(docs/05 §6) + 커버 이미지 업로드 + 실시간 미리보기 |
| 운영 | 영업시간(요일별), 주문 일시정지, `useInProgressStatus`(조리중 단계 사용), `prepayRequired`(선결제 필수), 재알림 간격 |
| 직원 | 초대(이메일)·역할 변경·비활성화 (OWNER) |
| 구독 | 현재 플랜·한도 사용량 게이지·결제수단·플랜 변경 (OWNER, docs/09) |

## 8. 태블릿 상주 모드

- PWA(manifest + 홈 추가 안내), Wake Lock API로 화면 꺼짐 방지 토글, 전체화면 모드.
- 세션 만료 대응: 스태프 세션은 장기 리프레시(30일), 401 시 PIN 재잠금 화면(백로그) 대신 MVP는 재로그인 유도.
- 오프라인: 배너 표시 + 조작 차단(낙관적 업데이트로 인한 유실 방지), 복구 시 보드 전체 재동기화.

## 9. 수용 기준 하이라이트 (qa 시나리오 씨앗)

- A-1 주문 생성 후 3초 내 보드에 카드+알림음 (S-5)
- A-2 `확인` 탭 → 고객 현황 타임라인이 5초 내 갱신 (C-5)
- A-3 품절 토글 → 룩북 피드에서 즉시 SOLD OUT 처리, 신규 주문 시도 시 서버 거부 (C-9/I-3)
- A-4 부분 선결제 세션의 카운터 정산 금액 = 총액 − PG결제액 (I-5)
- A-5 실시간 채널 강제 단절 → 10초 내 폴백 표시등 + 신규 주문 폴링 수신 (N-4)
- A-6 재료 입력→AI 생성→후보 선택→룩북 디테일 컷 반영까지 운영자 단독 3분 내 완료 (S-13, 리허설 기준)
