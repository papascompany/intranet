### 1.2 선정: 본선 **연남·연희**, 백업 **성수**

| 판단 | 논리 |
|---|---|
| 연남·연희 = 본선 | ICP 정합 최고: 8~20T 오너 직영 브런치·베이커리가 골목 단위 밀집 — 태블릿 경제성이 안 나오는 전형(docs/13 §1). 골목 밀집으로 도보 스카우팅·재방문 효율 최고(퍼널 상단 볼륨). DM·방문의 수신자가 오너 본인일 확률이 높아 의사결정이 빠름 |
| 성수 = 백업 | 브랜딩 화제성·사례 파급력 1위지만 대형화·기업형 매장 비중 증가로 ICP 밀도 희석(추정), 제휴 DM 피로도·게이트키퍼(매니저/대행) 리스크. 베타 사례 확보 후 1단계 확장 1순위 상권 |
| 뒤집기 게이트 | 스카우팅 결과 연남·연희의 태블릿 미도입 ICP 숏리스트가 15곳 미만이면 성수로 전환 |

- 망원: 총점 동률이나 인스타 파급력·객단가가 반 발 아래(추정) — 1단계 인접 확장 후보로 보존. 한남·이태원: 다이닝·와인바 비중이 높아 2단계 업종 확장(docs/13 §2)과 겹침 — 베타 제외.

### 1.3 하지 않는 것

- 실명 매장 리스트를 데스크에서 미리 만들지 않는다(추정 실명 나열 금지). 후보는 아래 절차로 **공개 데이터·현장에서만** 수집해 §6 트래커에 기입한다.

### 1.4 후보 발굴 절차 (롱리스트 40곳 → §2 스코어카드 → 숏리스트 15곳)

| 채널 | 방법 |
|---|---|
| 인스타 해시태그 | `#연남브런치` `#연남카페` `#연희동카페` `#연남베이커리` `#연남신상카페` 최근 게시물 상위 ~200개 → 매장 계정 추출 → 팔로워·피드 퀄리티·최근 게시일 기록 |

## 2. 매장 스크리닝 스코어카드

### 2.1 필수 게이트 (하나라도 미충족 시 제외)

| # | 게이트 | 확인 방법 |
|---|---|---|
| G1 | 테이블 8~20개 | 방문·리뷰 사진 카운트 |
| G2 | 태블릿 오더 미도입(타사 약정 잔존 없음) | 방문 목격 + 미팅 질문 |
| G3 | 인스타 계정 운영 중(최근 90일 게시물 ≥ 8) | 계정 확인 |
| G4 | 오너가 의사결정권자(직영·비프랜차이즈) | 미팅 확인 |

### 2.2 가점 스코어 (항목당 0~2점, 12점 만점 — **합격선 8점**, 가설)

| 항목 | 2점 기준 |
|---|---|
| 피드 사진 퀄리티 | 자연광·플레이팅·보정 톤 일관 — 촬영 투자 흔적 뚜렷(룩북 '감성'은 사진 품질에 의존, docs/10 리스크 1) |
| 인스타 팔로워 | ≥ 3,000 (1,000~2,999 = 1점, 가설) |
| 신메뉴 주기 | 분기 1회 이상 시즌·신메뉴 릴리즈(촬영·AI 연출컷 반복 수요) |
| 브랜딩 지출 흔적 | 로고·패키지·공간·유니폼 중 2개 이상 |
| 홀 운영 구조 | 피크에 홀 전담 1~2명 + POS 보드용 태블릿/PC 보유 또는 구매 의사 |
| 디지털 수용도 | 스토리 운영·예약/웨이팅 툴 등 신규 도구 사용 이력 |

### 2.3 피해야 할 매장 시그널

- 회전율 극대화형(줄 세우고 빼는 구조) — 탐색형 룩북과 상극, 'QR이 태블릿에 밀린' 세그먼트(docs/13 §1)
- 태블릿 오더 약정 잔존 — 스위칭 대상이 아니다(docs/01 §2)
- 위탁·매니저 운영으로 오너 부재 — 주 1회 피드백 루프 불성립
- 촬영·사례 공개 거부감 — 패키지의 교환 조건 자체가 불성립
- 지하 등 통신 음영 — 손님 LTE 불량이면 QR 진입부터 실패
- 폐업 임박 시그널(영업시간 축소, 리뷰 급감, 임대 문의 부착)
- 홀에 POS 보드 둘 기기·자리가 없고 마련 의사도 없음(주방 전달 리스크 — docs/10 리스크 표)

1. 데모 매장 QR 스캔 → 3초 진입(로그인 없음) 시연
2. 커버→스프레드 스크롤 — "사장님 사진이면 이렇게 보입니다" (감상 5초, 말 끊지 않기)
3. 퀵담기 3탭으로 주문 전송
4. 지참 태블릿 POS 보드에서 알림음과 함께 수신 — "홀에서는 이 화면만 보면 됩니다"
5. 주문 현황 `한 번 더` 재주문 — "객단가를 올리는 장치입니다(목표 세션당 추가주문 25%)"
6. 마무리: 촬영 패키지 조건(§3) 1장 요약 전달 → 2차 데모 또는 협약 일정 약속

### 4.3 예상 반론 5 + 대응

| 반론 | 대응 |
|---|---|
| "태블릿 오더 검토 중인데" | 목적이 다르다: 태블릿=회전 단말, 우리=브랜드 지면+객단가. 15테이블 태블릿 임대 총액 vs 월 ₩29,000 환산 비교(docs/09 §3 — 경쟁 수치는 docs/13 §8 검증 후 인용). 약정 0·하드웨어 0·당일 개통이니 태블릿 결정 전 8주 무료로 실측해 보시라 |
| "QR 주문은 실패했다던데" | 세그먼트 문제로 읽는다(docs/13 §1): 회전율 중심 대형 매장 이야기이고, 브런치처럼 머무는 매장이 우리 타깃. 종이 메뉴판 병행 유지 + 스캔→주문 전환율을 대시보드로 함께 실측하는 조건 제안 |
| "메뉴 입력 귀찮다" | 입력은 우리가 한다(룩북 세팅 대행 §3.1) — 사장님은 30분 검수만. CSV 일괄 임포트(docs/09 §2), 목표 '메뉴 30개 30분 이관'(docs/13 §4) |
| "수수료 떼가는 거 아니냐" | 주문 수수료 0·결제 경유 0, 수익은 월 구독뿐이고 베타는 무료. 결제·세무는 기존 포스 그대로 — '주문=MENUBOOK, 결제=기존 포스' 1페이지 가이드 제공(docs/13 §4) |
| "어르신·기계 어려운 손님은?" | QR은 추가 채널 — 직원 주문·종이 메뉴판 병행 그대로. 목차(차림 인덱스)가 사진 없는 전 메뉴 리스트라 목적형 손님도 빠름(docs/05 §4.3). 직원 호출 버튼 상시 |

### 4.4 퍼널 목표·주차별 활동량 (베타 개시일 기준 역산, 전환율 가설)

| 단계 | 목표 | 전환 가설 |
|---|---|---|
| DM 발송 | 100 | — |
| 미팅(방문 10분) | 20 | DM→미팅 20% |
| 데모(풀 시연) | 10 | 미팅→데모 50% |
| 계약(협약 서명) | **3 + 대기 1** | 데모→계약 30% — 계약은 '판매'가 아니라 §2 스코어 상위부터 '선발' |

| 주차 | 활동량 |
|---|---|
| W-4 | 스카우팅 반나절×2 + 롱리스트 40 완성, DM 25건 |
| W-3 | DM 25건 + 팔로업, 미팅 5건 |
| W-2 | DM 25건, 미팅 8건, 데모 4건, 계약 1호 확약 |
| W-1 | DM 25건, 미팅 7건, 데모 6건, **계약 3 + 대기 1 확약**, 촬영·온보딩 일정 확정 |

- 착수 게이트: 스카우팅은 즉시 가능. DM·데모는 데모 매장이 실기기에서 화보급으로 보이고 주문 체험이 되는 시점(M3 완료) 이후. 개시는 M6 DoD 통과 후(docs/10) — 개발이 밀리면 W-4~W-1도 함께 민다.
- 주간 실측으로 전환율 교정: DM→미팅 10% 미만이면 첫 문장 A/B 후 재개.

### 5.2 측정 방법 — 추가주문율 25%의 실측 정의

- **정의**: 세션당 추가주문율 = 2번째 이후 주문이 1건 이상 발생한 테이블 세션 수 ÷ 주문 1건 이상 발생한 테이블 세션 수 (세션 = TableSession 착석~정산, docs/03).
- **제외**: 직원 테스트 주문(스태프 deviceId 태깅), 전량 거절·취소 세션, 개시 후 첫 3일(운영 미숙 구간, 가설).
- **판정 표본**: 매장당 누적 유효 세션 ≥ 200(가설) 도달 후 판정. 보조 지표: 스캔→첫주문 ≥ 55%, 추가주문 기여 객단가(위젯).
- **확인 절차(매주 월)**: ① POS 통계의 객단가·추가주문 기여 위젯(docs/09 §3) ② 슈퍼어드민 매장 지표 스냅샷 교차 확인 ③ §6 트래커에 주차별 기록 → 주간 미팅에서 오너와 함께 열람(ROI 증명 = 리텐션 선행지표, docs/13 §7).

### 5.3 주간 피드백 질문지 (20분, 5문항 고정)

1. 이번 주 MENUBOOK 때문에 편해진 것 하나, 불편해진 것 하나는?
2. 손님이 헤매거나 결국 직원을 부른 장면이 있었나? (화면·상황·연령대)
3. 주문 누락·오주문·알림을 못 들은 일이 있었나? (있었다면 당시 상황)
4. 메뉴판(사진·순서·문구)에서 바꾸고 싶은 것은?
5. 오늘 기준, 월 ₩29,000/₩59,000이라면 계속 쓰시겠나? 이유는? (W6부터 심화 인터뷰 연결)

### 5.4 중단·피벗 기준

| 트리거 | 조치 |
|---|---|
